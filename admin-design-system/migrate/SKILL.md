# /migrate - @l-fin/ui-components Migration

Migrate to @l-fin/ui-components design system.

## Output Language

**IMPORTANT**: All terminal output and user-facing messages MUST be in Korean (한국어).

## Migration Modes

### Mode A: Add Library to Existing Project

Install @l-fin/ui-components via npm to existing project.

```bash
/migrate                    # 프로젝트 분석 후 마이그레이션
/migrate src/components     # 특정 폴더만 마이그레이션
```

### Mode B: Plan File Based Migration (prev folder)

Migrate existing project from `prev/` folder in `create-lfin-app` generated project.
Uses plan file generated by `/migrate-plan` skill.

```bash
/migrate-plan                       # 1. prev/ 분석 → MIGRATION_PLAN.md 생성
/lfin:migrate                       # 2. 계획 기반 마이그레이션 실행
```

**Mode B Workflow**:

```
npx create-lfin-app my-app
    ↓
Copy existing project to prev/ folder
    ↓
Run /migrate-plan → Generate MIGRATION_PLAN.md
    ↓
Run /lfin:migrate
    ↓
Migrate prev/ code to src/
```

## Prerequisites

Before starting migration, check the following requirements.

**IMPORTANT**: For Mode B (`--plan` option), skip P-2 (GitHub Packages auth) since `create-lfin-app` projects are already configured.

### P-1. Node.js Version Check

Run the following command to check Node.js version:

```bash
node -v
```

**Requirement**: Node.js 22 or higher

If version is below 22:

```
❌ Node.js 22 버전 이상이 필요합니다.
현재 버전: {current_version}

Node.js 22 이상을 설치한 후 다시 시도해주세요.
- nvm: `nvm install 22 && nvm use 22`
- brew: `brew install node@22`
- 공식 사이트: https://nodejs.org/
```

**IMPORTANT**: Exit migration if Node.js version is below 22.

### P-2. GitHub Packages Authentication Check

Follow [references/common/github-packages-auth.md](references/common/github-packages-auth.md) to verify authentication.

**IMPORTANT**: Exit migration if GitHub Packages authentication is not configured.

## Procedure

### Step 0: Check for Plan File

Check if `MIGRATION_PLAN.md` exists in the project root:

```bash
Read MIGRATION_PLAN.md
```

1. **Read Plan File**

   ```bash
   Read MIGRATION_PLAN.md
   ```

2. **Parse Plan File**

   Extract from plan file:
   - Project overview (framework, router)
   - Detected libraries table
   - Folder structure mapping (prev → src)
   - Migration target file list
   - Migration order

3. **Verify prev Folder**

   ```bash
   ls prev/
   ```

   Error if prev folder missing:

   ```
   ❌ prev 폴더가 존재하지 않습니다.
   계획 파일 기반 마이그레이션은 prev/ 폴더가 필요합니다.
   ```

4. **Show Plan Summary and Get User Confirmation**

   Show plan summary and get user confirmation:

   ```markdown
   ## 📋 마이그레이션 계획 요약

   ### 프로젝트 정보

   | 항목       | 값                    |
   | ---------- | --------------------- |
   | 프로젝트명 | {name from plan}      |
   | 프레임워크 | {framework from plan} |

   ### 마이그레이션 대상

   | 카테고리            | 현재 | 대상 |
   | ------------------- | ---- | ---- |
   | {from plan file...} |

   ### 예상 작업량

   | 카테고리            | 파일 수 |
   | ------------------- | ------- |
   | {from plan file...} |
   ```

   Use `AskUserQuestion` tool:

   ```
   질문: "위 계획대로 마이그레이션을 진행할까요?"
   옵션:
   - "전체 진행" - 계획된 모든 항목 마이그레이션
   - "선택적 진행" - 마이그레이션 항목 선택
   - "취소" - 마이그레이션 중단
   ```

   **IMPORTANT**: Do NOT proceed without user confirmation.

5. **Skip Step 1, 2**

   Plan file already contains analysis results, so skip Step 1 (Project Analysis) and Step 2 (Detect Migration Targets), go to Step 3.

6. **Execute Plan-based Migration**

   Follow "Migration Order" section in plan file:

   ```markdown
   ## 5. 마이그레이션 순서

   1. **라우팅 설정** - TanStack Router 라우트 파일 생성
   2. **레이아웃 마이그레이션** - app/layout.tsx → routes/\_\_root.tsx
   3. **페이지 마이그레이션** - 각 page.tsx → routes/\*.tsx
   4. **컴포넌트 마이그레이션** - UI/아이콘/폼 변환
   5. **정리** - prev/components/ui/ 삭제, 미사용 패키지 제거
   ```

   **File Copy and Transform**:
   - Copy files from `prev/` to `src/` while transforming
   - Reference "Folder Structure Mapping" section in plan file
   - Reference "Migration Target Files" section in plan file
   - Use `https://admin-template.lpin.io/llms/*.md` files for component mapping (WebFetch)

**IMPORTANT**: When using `--plan` option:

- Skip P-2 (GitHub Packages auth) - already configured by create-lfin-app
- Skip Step 1, 2 and start from Step 3

### Step 1: Project Analysis (Mode A only)

1. **Check package.json dependencies**
   - Read package.json to identify dependencies and devDependencies

2. **Detect Package Manager**

   Check for lock files to determine package manager:

   | Lock File           | Package Manager | Install Command | Run Command |
   | ------------------- | --------------- | --------------- | ----------- |
   | `pnpm-lock.yaml`    | pnpm            | `pnpm add`      | `pnpm`      |
   | `yarn.lock`         | yarn            | `yarn add`      | `yarn`      |
   | `package-lock.json` | npm             | `npm install`   | `npm run`   |
   | `bun.lockb`         | bun             | `bun add`       | `bun`       |

   Also check `packageManager` field in package.json (e.g., `"packageManager": "pnpm@10.27.0"`).

3. **Detect Framework**

   Check dependencies in package.json to determine framework:

   | Dependency               | Framework | Config File                           | Notes           |
   | ------------------------ | --------- | ------------------------------------- | --------------- |
   | `next`                   | Next.js   | `next.config.js/ts/mjs`               | SSR/SSG support |
   | `react` (without `next`) | React     | `vite.config.ts`, `webpack.config.js` | SPA             |
   | `gatsby`                 | Gatsby    | `gatsby-config.js`                    | Static site     |
   | `remix`                  | Remix     | `remix.config.js`                     | Full-stack      |

## Arguments

- `--plan <file>`: Plan file based migration (Mode B). Use MD file from `/migrate-plan`
- `--analyze-only`: Only analyze, don't execute migration (Mode A only)
- `--dry-run`: Preview changes only
- `<path>`: Target specific file/folder (e.g., `src/components`)

## Examples

### Mode A: Add Library to Existing Project

```
/migrate                    # Full project migration
/migrate --analyze-only     # Analysis only
/migrate src/pages          # Specific folder only
```

### Mode B: Plan File Based Migration (prev → src)

```
/migrate-plan                       # Step 1: Analyze prev/ → Generate MIGRATION_PLAN.md
/lfin:migrate                       # Step 2: Execute plan-based migration
```

## Notes

### Common

- Verify git status is clean before migration
- Run Step 8 verification after each migration phase
- Use `git checkout .` to rollback if needed

### Mode A (Existing Project)

- Always perform fresh analysis for each run
- Keep existing project structure, only replace libraries

### Mode B (Plan File Based)

- Must generate plan file with `/migrate-plan` first
- Review and edit plan file before migration recommended
- Move code from `prev/` to `src/` while transforming
- Can delete `prev/` folder after migration complete
