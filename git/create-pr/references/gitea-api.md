## Authentication

Global security — any of these methods:
- `AccessToken` — apiKey in query `access_token`; This authentication option is deprecated for removal in Gitea 1.23. Please use AuthorizationHeaderToken instead.
- `AuthorizationHeaderToken` — apiKey in header `Authorization`; API tokens must be prepended with "token" followed by a space.
- `BasicAuth` — basic
- `SudoHeader` — apiKey in header `Sudo`; Sudo API request as the user provided as the key. Admin privileges are required.
- `SudoParam` — apiKey in query `sudo`; Sudo API request as the user provided as the key. Admin privileges are required.
- `TOTPHeader` — apiKey in header `X-GITEA-OTP`; Must be used in combination with BasicAuth if two-factor authentication is enabled.
- `Token` — apiKey in query `token`; This authentication option is deprecated for removal in Gitea 1.23. Please use AuthorizationHeaderToken instead.
